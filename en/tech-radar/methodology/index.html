Tech Radar Methodology

How I build and publish the Tech Radar each week — sources, frequency, selection and limits.

Why this Tech Radar

The Cloud / DevOps / AI ecosystem moves too fast to keep up with passive reading. The goal is twofold: forcing myself to look beyond my daily perimeter on a regular basis, and offering a short, structured read for anyone who wants to follow the essentials without drowning in noise.

The Tech Radar publishes around twenty selected articles every week, summarized in English and French, organized by topic. Not for endless scrolling — for a quick, up-to-date view of what matters.

Monitored sources

27 RSS / Atom feeds, picked to cover the topics I work on (Cloud, DevOps, AI) plus a bit of business and French tech news for context. The list evolves: a source that turns out to add little gets removed, a new one comes in when it brings real value.

  • Cloud: AWS News Blog, AWS DevOps Blog, Azure Blog
  • DevOps / Cloud Native: Kubernetes Blog, CNCF Blog, HashiCorp Blog, Docker Blog, Grafana Blog
  • CI/CD & platforms: GitHub Blog, GitLab Blog
  • AI / ML: Hugging Face Blog, Simon Willison's Weblog
  • Security: The Hacker News, BleepingComputer
  • Languages: Go Blog, Rust Blog, Real Python, Python Insider
  • Business / tech news: TechCrunch, Ars Technica, The Verge, FrenchWeb
  • General IT: The Register, Le Monde Informatique, Next, ZDNet France, LinuxFr

Frequency

Collection runs automatically every Saturday morning (UTC) via a GitHub Action. I review and validate manually before going live, so the actual publication time depends on my availability — usually Saturday during the day or Sunday.

Selection method

From ~200 to 400 raw articles collected to the twenty published, the pipeline looks like this:

  27 RSS / Atom feeds
         |
         |
  Automatic collection (Saturday morning, UTC)
  ~200 to 400 raw articles (last seven days)
         |
         |
  AI-assisted selection (Claude Haiku 4.5)
  Top 40 -> deduplication -> ~20 final articles
         |
         |
  FR + EN summaries, categorization
         |
         |
  HTML generation (edition + cards + nav)
         |
         |
  Pull Request -> manual review -> publication

The AI prioritizes substantial technical articles (releases, RFCs, post-mortems, deep dives), removes duplicates when several sources cover the same news, and skips purely promotional content or "10 tools for…" listicles.

AI disclosure

I use Claude Haiku 4.5 (Anthropic) for four tasks:

  • Selecting the most relevant articles among the ones collected
  • Detecting and merging duplicates (same news covered by multiple sources)
  • Writing the English and French summaries
  • Assigning a category to each article

The original sources and URLs are kept as is, without modification. The translated titles and summaries are AI-generated, then reviewed before publication. If you ever spot an inaccurate summary, the original article link is always there — that's the source of truth.

Categories

Six categories, picked to cover my professional spectrum without diluting attention into topics that are too far from it:

  • Business: funding, acquisitions, regulation, ecosystem — understanding how the market evolves.
  • Cloud: AWS, Azure, GCP, multi-cloud — my core area.
  • DevOps: CI/CD, IaC, Kubernetes, observability — daily tooling.
  • Tech: major releases, languages, developer platforms — general tech literacy.
  • Security: CVEs, breaches, supply chain, hardening — non-negotiable.
  • AI: models, infrastructure, MLOps — the field moves too fast not to follow it.

Limits & guarantees

This Tech Radar is a personal watch, not journalism. I pick sources I consider reliable and review every edition before publishing, but I don't independently fact-check. Despite the AI filtering and the review, partial duplicates, promotional content or listicle-style articles can occasionally slip through. When in doubt, the link to the original source is always there — that's what counts.

The pipeline source code is public: github.com/TuroTheReal/weekly-tech-radar.
