Méthodologie du Tech Radar

Comment je construis et publie le Tech Radar chaque semaine — sources, fréquence, sélection et limites.

Pourquoi ce Tech Radar

L'écosystème Cloud / DevOps / IA bouge trop vite pour être suivi en lecture passive. Mon objectif est double : me forcer à regarder régulièrement ce qui se passe au-delà de mon périmètre quotidien, et proposer une lecture courte et structurée à ceux qui veulent suivre l'essentiel sans se perdre dans le bruit.

Le Tech Radar publie chaque semaine une vingtaine d'articles sélectionnés, résumés en français et en anglais, classés par thème. Pas pour scroller pendant des heures : pour avoir une vue rapide et à jour de ce qui compte.

Sources surveillées

27 flux RSS / Atom, choisis pour couvrir les sujets sur lesquels je travaille (Cloud, DevOps, IA) ainsi qu'un peu d'actualité business et tech française pour le contexte. La liste évolue : une source peu utile finit par sortir, une nouvelle entre quand elle apporte quelque chose.

  • Cloud : AWS News Blog, AWS DevOps Blog, Azure Blog
  • DevOps / Cloud Native : Kubernetes Blog, CNCF Blog, HashiCorp Blog, Docker Blog, Grafana Blog
  • CI/CD & plateformes : GitHub Blog, GitLab Blog
  • IA / ML : Hugging Face Blog, Simon Willison's Weblog
  • Sécurité : The Hacker News, BleepingComputer
  • Langages : Go Blog, Rust Blog, Real Python, Python Insider
  • Business / actualité tech : TechCrunch, Ars Technica, The Verge, FrenchWeb
  • Généraliste IT : The Register, Le Monde Informatique, Next, ZDNet France, LinuxFr

Fréquence

La collecte se déclenche automatiquement chaque samedi matin (UTC) via une GitHub Action. Je relis et valide manuellement avant la mise en ligne, donc l'heure de publication varie selon ma disponibilité — généralement dans la journée du samedi ou le dimanche.

Méthode de sélection

De ~200 à 400 articles bruts collectés à la vingtaine publiée, le pipeline ressemble à ceci :

  27 flux RSS / Atom
         |
         |
  Collecte automatique (samedi matin, UTC)
  ~200 a 400 articles bruts (sept derniers jours)
         |
         |
  Selection assistee par IA (Claude Haiku 4.5)
  Top 40 -> deduplication -> ~20 articles finaux
         |
         |
  Resume FR + EN, categorisation
         |
         |
  Generation HTML (edition + cards + nav)
         |
         |
  Pull Request -> validation manuelle -> publication

L'IA choisit en priorité les articles techniques substantiels (releases, RFC, post-mortems, deep dives), élimine les doublons quand plusieurs sources couvrent la même actu, et évite les contenus purement promotionnels ou les listes "10 outils pour…".

Divulgation IA

J'utilise Claude Haiku 4.5 (Anthropic) pour quatre tâches :

  • Sélectionner les articles les plus pertinents parmi ceux collectés
  • Détecter et fusionner les doublons (même actualité couverte par plusieurs sources)
  • Rédiger les résumés en français et en anglais
  • Attribuer une catégorie à chaque article

Les sources et URLs originales sont conservées telles quelles, sans modification. Les titres traduits et les résumés sont générés par l'IA, puis relus avant publication. Si tu trouves un résumé incorrect, le lien vers l'article original est toujours présent — c'est lui qui fait foi.

Catégories

Six catégories, choisies pour couvrir mon spectre professionnel sans noyer l'attention dans des sujets trop éloignés :

  • Business : levées, acquisitions, régulation, écosystème — comprendre comment le marché évolue.
  • Cloud : AWS, Azure, GCP, multi-cloud — mon cœur de métier.
  • DevOps : CI/CD, IaC, Kubernetes, observabilité — les outils du quotidien.
  • Tech : releases majeures, langages, plateformes développeur — culture générale.
  • Sécurité : CVEs, breaches, supply chain, durcissement — incontournable.
  • IA : modèles, infrastructure, MLOps — la discipline évolue trop vite pour ne pas la suivre.

Limites & garanties

Ce Tech Radar est une veille personnelle, pas un travail journalistique. Je sélectionne des sources que j'estime fiables et je relis chaque édition avant publication, mais je ne vérifie pas les faits indépendamment. Malgré le filtrage IA et la relecture, des doublons partiels, des contenus promotionnels ou des articles de type "liste" peuvent ponctuellement passer. En cas de doute, le lien vers la source originale est toujours présent — c'est elle qui fait référence.

Le code du pipeline est public : github.com/TuroTheReal/weekly-tech-radar.
